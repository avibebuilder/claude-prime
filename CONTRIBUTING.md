# Contributing to Claude Prime

Thanks for your interest in contributing! Here's how to get involved.

## Ways to Contribute

- **Report bugs** — Open an issue describing what happened and steps to reproduce
- **Suggest skills** — Have an idea for a new workflow or domain skill? Open an issue to discuss
- **Submit skills** — Build and submit new skills or starter kits
- **Improve docs** — Fix typos, clarify instructions, add examples
- **Share workflows** — Show how you use Claude Prime in your projects

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a branch: `git checkout -b my-contribution`
4. Make your changes
5. Test with Claude Code in a project
6. Push and open a pull request

## Submitting a Starter Skill Kit

Starter skills live in `.claude/starter-skills/` and are domain-specific knowledge packs (e.g., frontend, backend, Docker). They get copied into target projects during priming.

## Guidelines

- Keep skills focused — one skill, one job
- Follow existing patterns and structure
- Test your changes with Claude Code before submitting
- Write clear commit messages

## Code of Conduct

Be respectful. We're here to build useful tools together.

## Questions?

Open an issue or start a discussion. We're happy to help.
